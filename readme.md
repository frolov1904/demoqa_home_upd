# demoqa_home_upd

## Описание проекта

Этот проект предназначен для автоматизации тестирования сайтов [DemoQA](https://demoqa.com/) и [Herokuapp](https://the-internet.herokuapp.com/) с использованием **Selenium WebDriver** и **Pytest**.  
Проект реализует архитектуру на основе паттерна **Page Object** и покрывает автотестами различные пользовательские сценарии: формы, таблицы, модальные окна, CSS‑стили, работу с alert‑диалогами и динамическими элементами интерфейса.

## Основные возможности

- Базовые классы страниц для повторного использования методов.
- Компонентная архитектура для работы с элементами (WebElement).
- Проверка наличия и текста ключевых элементов на разных страницах.
- Переход между страницами, работа с формами, таблицами,сортировкой, модальными окнами и браузерными вкладками
- Валидация URL, взаимодействие со стилями и alert-окнами.
- Тесты, написанные согласно практикам Pytest.

## Структура проекта

```
demoqa_home/
```

## Как начать работу

1. Клонируйте репозиторий или создайте структуру вручную.
2. Установите зависимости командой:
    ```
    pip install -r requirements.txt
    ```
3. Убедитесь, что Chromedriver или другой драйвер Selenium находится в проекте или прописан в PATH.
4. Запустите тесты командой:
    ```
    pytest -s
    ```

## Особенности

- Используется паттерн **Page Object Model**.
- Поддержка работы с сайтами DemoQA и Herokuapp.

- Фикстуры WebDriver находятся в `conftest.py`.
